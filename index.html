Build your first super cool NLP application in 30 Mins using Google's Dialog flow.  

Using DialogFlow(V1) , Twilio, Slack and hook.io for microservice.

Before we start!

What are we building?

Send a text message with simple natural text instructions on a Slack Channel. The instructions are processed using DialogFlow API and analysing which twilio sends a text/voice message.

This application helps to understand how various components are connected and get started with building super cool things.

Before we start, my sincere credits to KR Varma of Kerala for amazing article and a blog post.

 What are the pre requisites?

A basic javascript understanding and a credit card to register with Twilio and Google Cloud Platform. More than all these you need a great enthusiasm to explore!

 What is in it for us?


* This application you build can be integrated with your team's slack channel and send a text /voice message when he is not responding.

* Auto updates configuered through internal webhook of slack can be made to send text messages.

*Since the dilogue flow sends messages with ML minor spelling differences or format errors can be handled easily. 

* We will quickly learn how to build such connected things.

2. Setting up Hook.io!

What is Hook io?

Hook io is a micro-service which gives you a webhook (url end point) to send requests and process it. For example, In the app we are building, we send requests to Hook.io and then we process it (Remove lunwanyted strings) send it to Dialog flow. Dilogflow in return sends a processed response with parameters. These parameteres are then sent to twilio to send a text message.  

  1. CREATE ACCOUNT- Go to hook.io and Sign Up! It is free. The catch is , Unless you pay your code can be accessed by anyone.
  2. CREATE A MICROSERVICE- You can write the code in different languages. If you are following this tutorial, let the sample code be there. We shall set up Twilio and Slack and come back and start with the code.
